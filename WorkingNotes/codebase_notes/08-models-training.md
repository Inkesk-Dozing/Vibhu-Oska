# 08 — Models & Training

## Three Model Components

```
Models/
├── sovereign_gpt/    ← Custom GPT — the primary reasoning engine
├── router/           ← Task classifier — speculative routing brain
└── reasoning/        ← QLoRA fine-tuning — domain adaptation pipeline
```

---

## Sovereign GPT

**What**: Decoder-only transformer built from PyTorch primitives. The primary inference engine for CHAT tasks.

**Architecture (default config):**

| Layer | Implementation | Detail |
|---|---|---|
| Embedding | `nn.Embedding` | vocab_size=4000, hidden=256 |
| Position | RoPE | Applied inside attention, not added to embedding |
| Attention | Multi-Head Causal | 8 heads, 32-dim per head, causal mask |
| Feed-Forward | SwiGLU | gate_proj + up_proj → silu activation × up_proj |
| Normalization | RMSNorm | Before attention + FFN (pre-norm) |
| Depth | 6 layers | Stacked transformer blocks |
| LM Head | Linear | Weight-tied to embedding.weight |
| Output | Logits → sample | Temperature + top-k + top-p sampling |

**Training:**
- Optimizer: AdamW (lr=3e-4, weight_decay=0.01)
- Scheduler: OneCycleLR (anneal from lr to lr/10)
- Gradient clipping: 1.0
- Corpus: self-seeded 200+ Q&A pairs + custom corpus.txt additions
- Command: `python -m Models.sovereign_gpt.train`

**BPE Tokenizer (SovereignBPETokenizer):**
- Custom BPE vocabulary trained on the same corpus
- Special tokens: `<pad>`, `<eos>`, `<unk>`
- Default vocab size: 4000 (scales to 16000+ for larger training runs)
- `encode(text)` → list of int ids
- `decode(ids)` → text string

**Checkpoints:** ✅ Present
- `Models/sovereign_gpt/checkpoints/sovereign_gpt.pt` (5.2MB)
- `Models/sovereign_gpt/checkpoints/tokenizer_vocab.json` (178KB)

**Scaling Notes:** The default 256 hidden dim / 6 layers / 4000 vocab is intentionally small for fast iteration. For meaningful language generation, target at minimum: hidden=512, layers=12, vocab=16000, trained on 10M+ tokens.

## Router Model (VibhuOskaRouter)

**What**: Compact encoder transformer with dual classification heads. Routes prompts to the right inference engine before generation is attempted.

**Architecture:**

| Component | Detail |
|---|---|
| Type | Encoder (no causal mask) |
| Layers | 3 transformer blocks |
| Hidden | 64 |
| Heads | 4 |
| Input | SovereignBPE tokenized prompt, padded to max_seq_len |
| Task head | Linear(64 → 5) → softmax → [CHAT, CODE, OS, DESIGN, IMAGE] |
| Target head | Linear(64 → 3) → softmax → [sovereign-gpt, vibhu-core, backup] |

**Training:**
- Synthetic dataset generated by `dataset_generator.py` (600+ labeled examples)
- Adam optimizer, CrossEntropyLoss on both heads
- 10 epochs with checkpoint saving
- Command: `python -m Models.router.train`

**Checkpoints:** ✅ Present (10 epoch checkpoints + best_router.pt)
- `Models/router/checkpoints/best_router.pt` (3MB)
- `Models/router/checkpoints/router_vocab.json` (140KB)

## QLoRA Fine-Tuning Pipeline

**What**: Adapts Qwen2.5-Coder-3B to Vibhu-Oska-specific behavior using Parameter-Efficient Fine-Tuning (PEFT/LoRA) with 4-bit quantization.

**Key parameters:**
```
Base model:   Qwen/Qwen2.5-Coder-3B-Instruct (HuggingFace, downloads on first run)
Quantization: 4-bit NF4 + nested double quant (bitsandbytes)
LoRA rank:    r=16, alpha=32
Dropout:      0.05
Target:       q_proj, k_proj, v_proj, o_proj, gate_proj, up_proj, down_proj
Batch:        micro=1, grad_accum=16 → effective batch=16
Optimizer:    AdamW + CosineAnnealingLR
VRAM needed:  ≥8GB (RTX 4060 optimized)
```

**Data pipeline:**
1. Load `Data/training/feedback/feedback_dataset.jsonl` (approved RLHF examples)
2. Load `Data/training/vibhu_oska_instructions.jsonl` (optional custom data)
3. If empty: fall back to built-in 2-example demo dataset (produces very low quality — add real data)

**Training loop:**
- Gradient accumulation (16 steps before optimizer.step())
- Gradient clipping at 1.0
- Per-epoch checkpoint saved to `Models/reasoning/lora_adapters/epoch_N/`
- Final adapter saved to `Models/reasoning/lora_adapters/final/`

**Command:** `python -m Models.reasoning.finetune --model qwen2.5-coder --epochs 3`

**Output:** PEFT LoRA adapter (not full weights — only the small delta matrices). Load on top of the base model for inference:
```python
from peft import PeftModel
model = PeftModel.from_pretrained(base_model, "Models/reasoning/lora_adapters/final")
```
